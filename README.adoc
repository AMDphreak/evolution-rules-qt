= Evolution Rules: Code
:toc:
:icons: font

Evolutionary rules for source code migrations between library and framework versions, designed for the link:https://github.com/AMDphreak/evolution-engine[Evolution Engine].

== Ecosystem

* link:https://github.com/AMDphreak/evolution-engine[Evolution Engine] — The core binary that resolves and applies these rules.
* link:https://github.com/AMDphreak/evolution-engine-action[Evolution Engine Action] — Modular GitHub Action to automate code evolution in CI pipelines.
* link:https://github.com/AMDphreak/evolution-rules-code[Evolution Rules: Code] — This repository (PyQt, library-level migrations).
* link:https://github.com/AMDphreak/evolution-rules-filesystem[Evolution Rules: Filesystem] — Rules for cross-OS path mapping and equivalency.

== Overview

This repository contains SDL-based rulesets for evolving source code. Rules are organized by language and library binding to ensure precise and safe migrations.

Currently featuring comprehensive rules for **PyQt5 to PyQt6** migrations.

== Directory Structure

[source]
----
rules/
└── python/             <-- Language
    └── PyQt/           <-- Library/Binding
        └── 5.15-6.0.sdl  <-- Version Migration Rule
----

== Usage with Evolution Engine

To migrate a Python project from PyQt5 to PyQt6:

[source,bash]
----
evolution-engine -p . \
  --rules-repo https://github.com/AMDphreak/evolution-rules-code \
  --library python/PyQt \
  --from 5.15 --to 6.0 --in-place
----

== Contribution

New rules should be added to the appropriate `rules/<lang>/<library>/` subdirectory. Each rule file should be named according to the version jump (e.g., `1.0-2.0.sdl`).
